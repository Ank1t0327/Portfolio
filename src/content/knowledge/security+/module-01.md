## Quick Idea

Security is basically about protecting **data**, **systems**, and **people** from bad access, bad changes, and downtime.

## The CIA Triad

> EXAM TIP
> If a question says data was leaked, changed, or unavailable, map it to CIA.

### Confidentiality

Only the right people can see the data.

> Example: HR salary records are visible only to HR.

### Integrity

Data stays correct and untampered.

> Example: A bank record cannot be secretly changed.

### Availability

Systems are reachable when users need them.

> Example: A website stays online during heavy traffic.

| Concept | Meaning | Easy Control |
| --- | --- | --- |
| Confidentiality | Keep it private | Encryption |
| Integrity | Keep it correct | Hashing |
| Availability | Keep it online | Backups |

### Non-Repudiation

Users cannot deny actions they performed.

> Example: A digital signature proves who approved a file.

## Cybersecurity Framework Functions

Frameworks give security teams a simple workflow:

* **Identify:** Know assets and risks.
* **Protect:** Add safeguards.
* **Detect:** Spot suspicious activity.
* **Respond:** Take action during an incident.
* **Recover:** Restore normal operations.

## Gap Analysis

Compare where security is now with where it should be.

> Example: Company needs MFA, but only uses passwords. That missing MFA is the gap.

## Access Control

Access control answers: **who can access what?**

### IAM Flow

* **Identification:** Claim identity. Example: username.
* **Authentication:** Prove identity. Example: password, OTP, fingerprint.
* **Authorization:** Get permissions. Example: admin vs normal user.
* **Accounting:** Log activity. Example: login history and file access.

## Key Takeaways

> KEY TAKEAWAYS
> - CIA = private, correct, available.
> - Non-repudiation = cannot deny the action.
> - Framework flow = Identify, Protect, Detect, Respond, Recover.
> - Gap analysis = current state vs desired state.
> - IAM = identify, authenticate, authorize, account.
